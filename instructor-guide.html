TechFlow Digital - Instructor Guide

WCAG 2.2 Testing Guide

This guide helps instructors identify the subtle accessibility issues embedded throughout the TechFlow Digital website. The site appears to be a normal business website but contains carefully placed WCAG violations for educational purposes.

Overview

The TechFlow Digital website simulates a realistic digital agency with embedded accessibility issues. Unlike obvious testing sites, these issues are subtle and require actual testing to discover - just like real-world accessibility auditing.

Teaching Approach: Encourage participants to use the same testing methods they would use on real client websites: keyboard navigation, screen readers, contrast checkers, and automated tools.

Testing Instructions for Participants

Recommended Testing Approach

  1. Keyboard Navigation: Tab through entire site, check focus visibility and logical order
  2. Screen Reader Testing: Use NVDA, JAWS, VoiceOver, or ORCA to identify missing announcements
  3. Contrast Analysis: Use browser dev tools or contrast checker extensions
  4. Form Testing: Test all forms for proper labeling, error handling, and required field indicators
  5. Timing Testing: Look for auto-updating content, timeouts, and moving elements
  6. Zoom Testing: Test at 200% zoom for reflow and text scaling issues
  7. Automated Scanning: Run axe or WAVE to catch obvious issues

Teaching Notes

Realistic Simulation: These issues mirror real-world mistakes developers make. Emphasize that accessibility testing is an ongoing process, not a one-time check.
Progressive Disclosure: Start with obvious issues (keyboard navigation, contrast) then move to subtle ones (ARIA implementation, status messages).
Mixed Examples: Point out both correct and incorrect implementations. This website contains both - just like real sites where some things are accessible and others aren't.
